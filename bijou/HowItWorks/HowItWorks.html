How It Works

Well, it's pretty simple. Running bijou.exe simply recursively reads the content/ folder applies matching files from the templates/ folder and writes out the result in the site/ folder

  • content/
    • 1.Menu1/ - Folders prefixed with numbers create the navigation hierarchy
      • foo.html - Matches to the generic HTML template.
    • 2.FileList/
      • file1.html - multiple files of the same template create a <ul> list
      • file2.html
      • file3.html
    • 3.SectionList/
      • _section1.html - (underscore prefixed files) assembles the file sections in one file
      • _section2.html
      • _section3.html
    • 0.Invisible/ - Folders prefixed with '0.' are processed by the template engine but are excluded from the navigation hierarchy.
      • default.rss - Matches to the generic RSS template..
    • extra/ - Folders NOT prefixed with numbers are simply copied (and not processed via templates).
      • logo.png
      • main.js
      • styles.css
  • templates/ - Template files contain common markup repeated through the site and substitution placeholders for menu and content items.
    • default.html - HTML generic template (simple substitution of body content)
    • default.csv - CSV generic template (converts the CSV file to a table)
    • default.md - Markdown generic template (incomplete Markdown implementation)
    • default.rss - RSS Feed generic template (also produces an HTML version for the site)
    • default.ics - iCalendar generic template (also produces an HTML version for the site)
    • default.pics - picture generic template
    • default.song - song generic template
    • default.rcp - recipe generic template
    • default.slides - presentation slide generic template
  • site/
    • The end result. This is where your static HTML site is created.
