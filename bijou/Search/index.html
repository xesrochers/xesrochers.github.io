  • index Welcome to Bijou. Bijou is an experimental template based static website generator written in .NET Great for prototyping! Need to get a prototype going quick and fast. You probably want to consider bijou! Absolutely no scripting Well, that's a double edge sword. It make things simple, however with simplicity comes limitations. Maybe the limitations are not for you, so read on... Installation You really only need bijou.exe (and mono if you are running this on Linux). You might want to start off with the sample content/ and templates/ folders provided with this code. How to use it Well, it's pretty simple. Simply run bijou.exe at the root of a folder that contains: content/ a folder hiearchy with various content files templates/ a folder with processing templates site/ a destination folder where the processed files are written Inspiration I have to thank Ben Schwarz at Bonsai ( a PHP based version, with way more features I might add) as this is where I took a few of the ideas for this experimental project. Mucho thanks Ben! License Well this is really experimental code. Use at you own risk. Released under the MIT LICENSE. Copyright Sly Desrochers.
  • How It Works {$title} Well, it's pretty simple. Running bijou.exe simply recursively reads the content/ folder applies matching files from the templates/ folder and writes out the result in the site/ folder content/ 1.Menu1/ - Folders prefixed with numbers create the navigation hierarchy foo.html - Matches to the generic HTML template. 2.FileList/ file1.html - multiple files of the same template create a <ul> list file2.html file3.html 3.SectionList/ _section1.html - (underscore prefixed files) assembles the file sections in one file _section2.html _section3.html 0.Invisible/ - Folders prefixed with '0.' are processed by the template engine but are excluded from the navigation hierarchy. default.rss - Matches to the generic RSS template.. extra/ - Folders NOT prefixed with numbers are simply copied (and not processed via templates). logo.png main.js styles.css templates/ - Template files contain common markup repeated through the site and substitution placeholders for menu and content items. default.html - HTML generic template (simple substitution of body content) default.csv - CSV generic template (converts the CSV file to a table) default.md - Markdown generic template (incomplete Markdown implementation) default.rss - RSS Feed generic template (also produces an HTML version for the site) default.ics - iCalendar generic template (also produces an HTML version for the site) default.pics - picture generic template default.song - song generic template default.rcp - recipe generic template default.slides - presentation slide generic template site/ The end result. This is where your static HTML site is created.
  • Reference {$title} Command Line Options -h Used to display the help menu -w Used to trigger the watch feature which automatically monitors the file system for changes -i Used to activate index.html based urls (to browse site without a webserver) -c Used to clear/remove the 'site' folder (start fresh as old stuff could hang around in there) -o:folder Used specify an output folder (different than site) -r:folder Used specify the root path. Used when your files are not located at the website root -v Used to enable verbose mode -d Used to enable debug mode Filename underscore prefixed filenames (_section-file.html) assembles the file sections in one file dash in filenames (group-file.html) will bold (.group CSS) the group part of the file name in the display list

Will filter contents of $('.searchdata-js') Current value: not displayed {$searchdata} Used inject the search data in a search page. Designed for jquery.fastlivefilter.js Current value: not displayed {$children} Used inject links of the direct children of the page.
