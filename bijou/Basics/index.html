Basics

Well, it's pretty simple. Running bijou.exe simply recursively reads the content/ folder applies matching files from the templates/ folder and writes out the result in the www/ folder

content/ folder

The site content files. Where pages and sections are created.

1.Menu1/
Folders prefixed with numbers create the navigation hierarchy
foo.html - Matches to the generic HTML template.
2.FileList/
multiple files of the same template create a <ul> list
file1.html
file2.html
file3.html
3.SectionList/
underscore prefixed files. assembles the file sections in one file
_section1.html
_section2.html
_section3.html
0.Invisible/
Folders prefixed with '0.' are processed by the template engine but are excluded from the navigation hierarchy.
default.rss - Matches to the generic RSS template..
extra/
Folders NOT prefixed with numbers are simply copied (and not processed via templates). Folders such as css, javascript, fonts, images...
logo.png
main.js
styles.css

template/ folder

Template files contain common markup repeated through the site and substitution placeholders for menu and content items. The Section Processor is executed if no processor is activated

default.html - HTML generic template (simple substitution of body content)
default.csv - Kicks the CSV processor
default.md - Kicks the Markdown processor
default.rss - Kicks the RSS processor
default.song - Kicks the Song processor
default.rcp - Kicks Recipe processor

www/ folder

The end result. This is where your static HTML site is created.
